# Поддержка безопасности (Security Policy)

## Поддерживаемые версии

В настоящее время я выпускаю обновления безопасности только для актуальных (последних) версий проекта. Пожалуйста, убедитесь, что вы используете свежие коммиты / последние релизы.

| Версия проекта | Поддерживается     |
| -------------- | ------------------ |
| `main` ветка   | :white_check_mark: |
| Последний релиз| :white_check_mark: |
| Старые релизы  | :x:                |

## Как сообщить об уязвимости (Reporting a Vulnerability)

Я очень серьезно отношусь к безопасности утилиты. Если вы обнаружили уязвимость или проблему с безопасностью в `go-repo-orchestrator`, пожалуйста, **не создавайте публичный Issue**, чтобы избежать эксплуатации проблемы до публикации моего патча.

Вместо этого я прошу вас:
1. Использовать механизм **GitHub Security Advisories**: Перейдите на вкладку **Security** → **Advisories** в этом репозитории на GitHub и нажмите **"Report a vulnerability"** для отправки мне приватного отчета.
2. Альтернативно, вы можете написать мне напрямую (контакты доступны в моем GitHub-профиле).

**Процесс обработки:**
- Я постараюсь подтвердить получение информации об уязвимости в течение 48 часов.
- Я предоставлю вам оценку проблемы и примерные сроки выпуска исправления.
- После выпуска патча я, при вашем желании, с радостью упомяну вас в благодарностях при публичном раскрытии (Public Advisory).
